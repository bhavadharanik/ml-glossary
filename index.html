1. ML Fundamentals

How Machine Learning Works: Training Data ──> [ Model ] ──> Predictions │ ▲ │ │ │ │ └── features ──> Learning Algorithm <── loss (error) (gradient descent) Analogy: Like TDD but automated Input examples = test cases Model = your function Training = auto-fixing the function until tests pass Loss = number of failing tests
Machine Learning
A program that learns patterns from data instead of being explicitly programmed with rules.
>> Instead of writing if/else rules, you show the system 10,000 examples and it figures out the rules itself.
CRITICAL
Model
A mathematical function that takes input and produces a prediction. Has parameters (weights) that are learned during training.
>> Like an API endpoint with learned logic instead of hand-written logic. Input goes in, prediction comes out.
Training
Showing a model many examples so it adjusts its parameters to make better predictions. Minimizes a loss function.
>> Like running your function against thousands of test cases, auto-tweaking parameters each time until outputs match expected results.
Inference
Using a trained model to make predictions on new, unseen data. This is what happens in production.
>> Like calling your deployed API in production. Training = development, inference = production.
Loss Function
Measures how wrong the model's predictions are. MSE for regression, cross-entropy for classification. Goal: minimize this.
>> Like an error metric in your Grafana dashboard. Lower = better. Training tries to push this number down.
CRITICAL
Gradient Descent
Algorithm that adjusts model parameters to reduce loss. Calculates which direction to change each parameter and by how much.
>> Your model is wrong. The loss tells you HOW wrong. The gradient tells you WHICH DIRECTION to fix. The learning rate tells you HOW MUCH to fix. Like debugging with a compass.
CRITICAL
Overfitting
Model memorizes training data but fails on new data. High training accuracy, low test accuracy.
>> Like hardcoding "if userID == 123 return fraud". Works on test data, useless in production.
CRITICAL
Underfitting
Model is too simple to capture patterns. Low accuracy on both training and test data.
>> Like writing a single if/else rule when you need a complex decision tree. Too simple.
Supervised Learning
Training with labeled data: input AND correct answer. Classification and regression are supervised.
>> Like unit tests with expected outputs. The model learns to match input to correct output.
CRITICAL
Unsupervised Learning
Training without labels: model finds patterns/groups on its own. Clustering is unsupervised.
>> Like grouping log entries by pattern without pre-defining categories.
Train / Test / Validation Split
Training (70-80%): model learns. Validation (10-15%): tune hyperparameters. Test (10-15%): final evaluation. NEVER touch test during training.
>> Like dev/staging/production environments. Develop on dev, test on staging, final check is production.
Feature
An individual measurable property used as input to the model. Age, income, message_count are all features.
>> Like parameters in an API request. Each feature is one field the model uses to decide.
Feature Engineering
Creating new features from raw data. E.g., extracting "day_of_week" from a timestamp.
>> Like adding computed columns: price_per_sqft = price / sqft. Makes data more useful.
Elara
Hyperparameters
Settings YOU choose before training (learning rate, batch size, layers). Not learned by the model.
>> Like config values in your Docker environment variables. You set them, the system uses them.

(TP + TN) / Total. Percentage of correct predictions. MISLEADING with imbalanced data.
>> A fraud system that always says "not fraud" is 99% accurate but catches ZERO fraud.
CRITICALEEG Project
Precision
TP / (TP + FP). Of everything predicted positive, how many were actually positive? Optimize when false positives are costly.
>> Your API returns 10 fraud alerts. 8 were real. Precision = 80%. "How trustworthy are my alerts?"
CRITICAL
Recall (Sensitivity)
TP / (TP + FN). Of all actual positives, how many did we catch? Optimize when missing positives is costly.
>> 20 real frauds existed. You caught 8. Recall = 40%. "How many real problems did I detect?"
CRITICALElara flags
F1 Score
2 * (P * R) / (P + R). Harmonic mean of precision and recall. Use when you need balance.
>> Your EEG project proved this: CNN had high accuracy but F1=0.013 (useless). CLIP had F1=0.345 (works). 27x improvement.

AUC-ROC
Area Under the ROC Curve. Plots True Positive Rate vs False Positive Rate. 1.0 = perfect, 0.5 = random.
>> Like a performance graph showing how well your system separates signal from noise across all sensitivity settings.
Class Imbalance
When one class dominates (99% not-fraud, 1% fraud). Models predict majority class and look accurate but are useless.

3. Classical ML Models

When to use which model: Tabular data? ├── Need interpretability? ──> Decision Tree / Logistic Regression ├── Best accuracy? ──> XGBoost (industry standard) ├── Good default? ──> Random Forest └── Predict a number? ──> Linear Regression No labels? └── Group similar items? ──> K-Means Clustering Text/Images/Sequence? └── Neural Networks / Transformers
Linear Regression
Predicts continuous values: y = w1*x1 + w2*x2 + ... + b. Minimizes MSE. Assumes linear relationship.
>> y = mx + b but with many features. Predict house price from size, rooms, location.
Logistic Regression
Classification (despite the name). Linear regression + sigmoid to output probability 0-1. Threshold at 0.5.
>> Like your Elara sigmoid: takes any number, squashes to 0-1. Above 0.5 = positive class.
Elara sigmoid
Decision Tree
If/else splits on features: "age > 30?" then "income > 50K?" Uses Gini impurity to choose splits. Overfits easily.
>> Like a flowchart/switch statement. Easy to understand but memorizes data if too deep.
Random Forest
100-1000 decision trees on random data subsets. Final prediction = majority vote. Ensemble method. Resistant to overfitting.
>> Like 100 microservice instances, each seeing slightly different data, taking consensus. More reliable than one.
CRITICAL
XGBoost / Gradient Boosting
Trees built SEQUENTIALLY: each tree fixes errors of previous. Tree 2 predicts ERRORS of Tree 1. Most used model in industry & Kaggle.
>> Like iterative bug fixing: each deployment fixes bugs from the previous one. Each tree patches what the last got wrong.
CRITICAL

Random Forest (Bagging)

Trees trained INDEPENDENTLY on random subsets. Vote at the end. Good default, hard to mess up.

XGBoost (Boosting)

Trees trained SEQUENTIALLY, each fixing previous errors. Higher accuracy, needs more tuning.

4. Deep Learning & Neural Networks

Neural Network: Input Layer Hidden Layer 1 Hidden Layer 2 Output [feature 1] ──> [ neuron ] ──> [ neuron ] ──> [prediction] [feature 2] ──> [ neuron ] ──> [ neuron ] ──> [feature 3] ──> [ neuron ] ──> [ neuron ] Each connection has a weight Each neuron: activation( weights * input + bias ) Training loop: 1. Forward pass ──> get prediction 2. Calculate loss ──> how wrong? 3. Backward pass ──> compute gradients (backpropagation) 4. Update weights ──> weights -= learning_rate * gradient 5. Repeat 1000x
Neural Network
Layers of neurons. Input -> hidden layers -> output. Each neuron: weighted sum + activation function.
>> Like a multi-layer microservice pipeline. Data flows through each layer, getting transformed.
Backpropagation
After forward pass, calculate loss, then flow error BACKWARDS to compute how much each weight contributed. Then update weights.
>> Like distributed tracing in reverse: error traced back through each service to find which component caused it.
CRITICAL
ReLU (Rectified Linear Unit)
max(0, x). If positive, pass through. If negative, output 0. Most common activation for hidden layers.
ReLU(5) = 5, ReLU(-3) = 0, ReLU(0.5) = 0.5
Sigmoid
1 / (1 + e^(-x)). Squashes any number to 0-1. Used for binary classification output.
>> YOU KNOW THIS from Elara: converts z-scores into 0-100 health scores. Same function!
ElaraCRITICAL
Dropout
During training, randomly set 30% of neurons to 0. Forces resilience. Turned OFF during inference.
>> Like chaos engineering: randomly killing pods during testing. Disabled in production.
Transfer Learning
Using a model trained on one task as starting point for another. The pretrained model already knows useful features.
>> Your EEG project: CLIP trained on 400M internet images works on EEG spectrograms. Like reusing a payment service across projects.

5. NLP (Natural Language Processing)

Tokenization
Breaking text into tokens. "unhappiness" -> ["un", "happi", "ness"]. BPE most common. LLMs see tokens, not words.
>> Like splitting a URL into path segments. The model processes tokens, not raw strings.
Embedding
Dense vector representation. Similar items = similar vectors. "king" - "man" + "woman" ~ "queen". Typically 512-768 dimensions.
>> Like hashing but meaningful: similar inputs produce similar outputs. A 512-dim fingerprint of meaning.
CRITICAL
Cosine Similarity
Measures angle between two vectors. cos(A,B) = (A.B) / (|A|*|B|). 1.0 = identical, 0 = unrelated. Core of RAG retrieval.
cos([1,0], [0,1]) = 0 (perpendicular, unrelated) cos([1,0], [1,0]) = 1.0 (identical) cos([1,1], [1,0]) = 0.71 (somewhat similar)
CRITICAL

6. Transformers & Attention

Self-Attention: How Transformers Understand Context Sentence: "The cat sat on the mat because it was tired" For the word "it", attention looks at ALL other words: "The" -> low attention (0.02) "cat" -> HIGH attention (0.65) <-- "it" refers to "cat"! "sat" -> low attention (0.08) "mat" -> low attention (0.05) "tired" -> medium attention (0.20) This is how the model knows "it" = "cat" The Math: Q (Query) = "What am I looking for?" <-- like a WHERE clause K (Key) = "What do I contain?" <-- like a database index V (Value) = "What info do I provide?" <-- like the returned data Attention = softmax(Q * K^T / sqrt(d)) * V
Transformer
Architecture behind ALL modern LLMs. Processes all words simultaneously using self-attention. Replaced slow RNNs/LSTMs.
>> Every LLM you use (Claude at Elara, Gemini in NeutroFi) is a Transformer. THE architecture of modern AI.
CRITICAL
Self-Attention
Each word looks at every other word to understand context. Computed using Query, Key, Value vectors.
>> Like a service that checks ALL other services before responding, weighing which inputs matter most.
CRITICAL

Encoder (BERT)

Reads text bidirectionally. Creates rich representations. Best for: classification, NER, understanding.

Decoder (GPT/Claude)

Generates text one token at a time (left to right). Best for: text generation, conversation.

7. LLMs & Generative AI

How an LLM Generates Text: Input: "The cat sat on the" Step 1: Model sees all tokens, computes probabilities: "mat" 0.35 <-- highest "floor" 0.20 "bed" 0.15 "chair" 0.10 ... ... Step 2: Sample next token based on temperature: temp=0.0 -> always pick "mat" (deterministic) temp=0.7 -> usually "mat", sometimes "floor" temp=1.5 -> could pick anything (creative/random) Step 3: Append "mat", repeat from Step 1: "The cat sat on the mat" -> predict next -> "." (0.45) This is autoregressive generation: one token at a time
Large Language Model (LLM)
Very large transformer (billions of parameters) trained to predict next token on massive text data.
>> Claude Haiku (Elara) and Gemini (NeutroFi) are both LLMs. They just predict the next word, really fast, really well.
CRITICALElaraNeutroFi
Temperature
Controls randomness. 0 = deterministic (always pick highest probability). 1+ = creative/random. Divides logits before softmax.
>> Your NeutroFi uses temp=0.3 (focused). Higher = more creative but less reliable.
CRITICALNeutroFi
Prompt Engineering
Designing prompts for better output. Zero-shot (no examples), few-shot (with examples), chain-of-thought (step by step).
>> You do this at Elara: designing prompt templates for signal classification.
CRITICALElara
Hallucination
LLM generates plausible but factually incorrect information. A fundamental challenge.
>> This is why NeutroFi's risk manager uses rules, not LLM output. And why RAG grounds responses in real docs.
NeutroFi

Fine-tuning

Change model behavior. Needs training data + compute. Risk: catastrophic forgetting. Best for: specific style/format.

RAG

8. RAG & Agentic AI

RAG Pipeline (Retrieval Augmented Generation): OFFLINE (once): Documents ──> Chunk (500 tokens) ──> Embed (sentence-transformers) ──> Store in Vector DB (FAISS) ONLINE (per query): User Question ──> Embed ──> Search Vector DB (cosine similarity) ──> Top 3 chunks retrieved ──> Build prompt: [context + question] ──> LLM generates grounded answer NOTE: Elara does NOT use RAG. But you must know it for interviews.
RAG (Retrieval Augmented Generation)
Embed docs -> store in vector DB -> embed query -> find similar chunks -> pass to LLM -> grounded answer. Reduces hallucination.
>> Like searching Stack Overflow before answering a question, instead of guessing from memory.
CRITICAL
Vector Database
DB optimized for similarity search on embeddings. FAISS (local), Pinecone, ChromaDB (managed). Uses cosine similarity.
>> Like a database with "find similar" instead of exact match.
Agent
LLM that can DECIDE what to do and USE TOOLS. Not just text generation - taking actions based on reasoning.
>> Your NeutroFi: LLM decides to call CoinGecko API, gets data, then generates analysis.
CRITICALNeutroFi
LangGraph
Graph-based agent workflows. Nodes = steps, edges = connections, state = shared data. More flexible than chains.
>> Your NeutroFi: StateGraph with 6 nodes. AgentState TypedDict carries data between nodes.
NeutroFi
Tool Use / Function Calling
Define tools the LLM can call. LLM decides WHICH tool and with WHAT arguments. You execute and return results.

10. Math Essentials

Vector
A list of numbers: [3, 4, 5]. Embeddings are high-dimensional vectors (512-dim). Core of all ML.
[3, 4] is a 2D vector. A 512-dim embedding is just a list of 512 floats.
Dot Product
Multiply elements and sum: [1,2,3] . [4,5,6] = 4+10+18 = 32. Core of attention and similarity.
np.dot([1,2,3], [4,5,6]) = 32
Standard Deviation
How spread out values are from the mean. Used in z-score: z = (value - mean) / std.
>> You use this in Elara: baseline_std is how much signal counts typically vary.

13. Self-Quiz (Click to Reveal)

Test Yourself

What's the difference between precision and recall?
Precision = of my positive predictions, how many were correct? Recall = of all actual positives, how many did I catch? Precision cares about false positives, recall cares about false negatives.
Why was 76% accuracy bad in your EEG project?
The CNN predicted the majority class for everything. 76% of the data was one class, so always predicting that class = 76% accuracy but F1=0.013. It learned nothing. CLIP actually classified both classes: 24% accuracy but F1=0.345.
Explain RAG in 30 seconds.
Embed your documents and store in a vector database. When user asks a question, embed the question, search for similar document chunks, pass those chunks + question to an LLM, get an answer grounded in real data instead of hallucinated.
What does temperature do in LLM inference?
Controls randomness. Temperature=0 always picks the highest probability token (deterministic). Temperature=1+ adds randomness (creative). Technically: divides logits by temperature before softmax.
Why does your NeutroFi risk manager use rules instead of LLM?
LLMs can hallucinate. For financial recommendations, you need deterministic guardrails. If confidence < 0.55, force "Hold" regardless. New buyers need >0.8 confidence. You can't trust pure LLM output for money decisions.
Random Forest vs XGBoost?
Random Forest: trees trained INDEPENDENTLY on random subsets, vote at end (bagging). XGBoost: trees trained SEQUENTIALLY, each fixing errors of the previous (boosting). XGBoost usually more accurate but needs more tuning.
What is backpropagation?
After the forward pass (prediction), calculate the loss (error). Then flow the error BACKWARDS through the network using chain rule to compute how much each weight contributed to the error. Then update weights proportionally.
Why did you use z-score + sigmoid at Elara?
Z-score compares a signal against its baseline (how unusual is this?). But z-scores can be any number (-3 to +3). Sigmoid squashes them to 0-100 for managers to read easily. A manager sees "score: 72" not "z-score: 1.3".
Explain the Transformer attention mechanism.
Each word has Q (query), K (key), V (value) vectors. Compute attention scores: Q dot K for every word pair. Scale by sqrt(d). Softmax to get probabilities. Multiply by V for weighted output. This lets each word attend to all other words simultaneously.
When would you use fine-tuning vs RAG vs prompting?
Prompting: quick, no training, good for simple tasks. RAG: need company-specific data, no training needed, bottleneck is retrieval. Fine-tuning: need specific behavior/format, requires data + compute, risk of catastrophic forgetting.
